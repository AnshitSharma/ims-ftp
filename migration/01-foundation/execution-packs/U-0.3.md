# U-0.3 — Nestable transactions in removeComponent / deleteConfiguration
Concept: transaction ownership symmetry (prereq for ReplaceComponentCommand, audit R-4).
Pins baseline: yes. Invariants touched: INV-10, INV-11.

## Purpose
removeComponent (line 934) and deleteConfiguration (line 3603) call beginTransaction()
unconditionally and rollback() unguarded — they explode inside any wrapping transaction.
Adopt the ownTransaction pattern addComponent already uses.

## Outputs
Both methods safe to call inside an existing transaction; commit/rollback only when owner.
Also fix the deleteConfiguration not-found path: initialize `$components = [];` before the
`if ($configData)` block so `count($components)` cannot fatal (audit R-6, structural part only —
serial-accurate release is P6 work, do NOT attempt it here).

## Files Modified (1)
core/models/server/ServerBuilder.php — pattern for both methods:
```php
$ownTransaction = !$this->pdo->inTransaction();
if ($ownTransaction) { $this->pdo->beginTransaction(); }
...
if ($ownTransaction) { $this->pdo->commit(); }
...catch...
if ($ownTransaction && $this->pdo->inTransaction()) { $this->pdo->rollback(); }
```
