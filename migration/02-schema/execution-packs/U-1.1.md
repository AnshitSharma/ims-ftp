```sql
CREATE TABLE config_components (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  config_uuid CHAR(36) NOT NULL,
  component_type ENUM('chassis','motherboard','cpu','ram','storage','nic','hbacard','pciecard','riser','caddy','sfp') NOT NULL,
  inventory_table VARCHAR(32) NOT NULL,
  inventory_id BIGINT UNSIGNED NOT NULL,
  spec_uuid CHAR(36) NOT NULL,
  serial_number VARCHAR(191) NULL,
  parent_id BIGINT UNSIGNED NULL,
  slot_ref VARCHAR(64) NULL,
  added_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  added_by INT NOT NULL DEFAULT 0,
  removed_at DATETIME NULL,            -- soft-tombstone during dual-write window ONLY; hard-deleted at U-D.3
  UNIQUE KEY uq_inventory_once (inventory_table, inventory_id, removed_at),
  UNIQUE KEY uq_slot_occupancy (config_uuid, slot_ref, removed_at),
  KEY k_config (config_uuid), KEY k_parent (parent_id),
  CONSTRAINT fk_cc_config FOREIGN KEY (config_uuid) REFERENCES server_configurations(config_uuid),
  CONSTRAINT fk_cc_parent FOREIGN KEY (parent_id) REFERENCES config_components(id)
) ENGINE=InnoDB;
```
NOTE 1: config_uuid column MUST declare the same CHARACTER SET/COLLATE as
server_configurations.config_uuid — query information_schema first and write it explicitly.
NOTE 2: inventory FK is (inventory_table, inventory_id) soft (checked by orphan_report), because
ten inventory tables exist until their unification (out of scope; recorded in
PLAN_VERIFICATION_REVIEW F-6). A hard FK lands post-unification.
NOTE 3: MySQL treats NULLs as distinct in unique keys — removed_at NULL rows are the live ones and
uniqueness holds for them; tombstones don't collide. Document this in the seeder header.
Rollback file: DROP TABLE config_components; (allowed only per R-SCHEMA).
