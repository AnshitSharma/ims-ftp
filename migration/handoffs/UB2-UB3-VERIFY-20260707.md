## ⚠️ Accepted scope narrowing in equivalence_report (INV-8) — human should be aware
U-B.2 fixed two PRE-EXISTING equivalence_report bugs by narrowing the comparison:
serial_number is now only compared for `cpu` (the only type the legacy decoder round-trips), and
slot_ref only for `sfp`. Consequence: **slot placement equivalence for
pciecard/riser/hbacard/storage, and serials for all non-cpu types, are NOT verified by INV-8's
report today.** The root cause is the legacy `extractComponentsFromJson()` dropping those fields
(the same family as the RAM serial gap flagged in P1-PL-VERIFY). Either the legacy decoder gets
fixed (preferred; makes the report's coverage full again) or this narrowing is accepted as a
permanent limit — a decision for the human, ideally before P8 (read cutover) leans on
equivalence as its primary safety proof.
