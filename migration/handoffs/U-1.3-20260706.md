- `scripts/verify/schema_report.php`: **one deviation from the pack's stated file scope** ("Files
  Created/Modified: seeder + rollback; expected_schema.json extended. Nothing else") — had to add
  a small MariaDB-vs-MySQL JSON-type equivalence check (`isJsonValidColumn()` + one `if` in the
  data-type comparison). Reason: `config_events.payload` is declared `JSON`, but MariaDB stores
  `JSON` as an alias for `longtext` with a `CHECK (json_valid(...))` constraint — running the
  seeder on the scratch MariaDB instance made `information_schema.COLUMNS.DATA_TYPE` report
  `longtext`, not `json`, which would falsely RED on any MariaDB target even though the schema is
  exactly as specified. The fix checks `information_schema.CHECK_CONSTRAINTS` for a `json_valid`
  constraint on that column and accepts it as equivalent to a native `json` type. No other check
  logic changed; flagging this as a scope note rather than treating it as silently in-bounds,
  since the pack was explicit about "nothing else."
