## Findings

### F-6 — `target_state_test.php` fixture premise invalidated by U-B.4 (FIXED, test-only)

Asserted `source=json` for every component on the premise that `config_components` was
empty for fixture `06ea5abb`. The backfill made that false (13 live rows, ids
10017–10082), so `fromCurrent()` correctly takes the **rows** path — 15 assertions failed
against *correct* code. `TargetStateBuilder`'s own docblock (lines 21-22) already stated
the rows path becomes universal once the backfill lands.

Rewritten to clear the rows inside a rolled-back transaction (leaf-first — `fk_cc_parent`
is RESTRICT and the tree is two deep) so **both** paths are now exercised deterministically.
Synthetic `inventory_id`s are offset by 900000 because `uq_inventory_once` is
`(inventory_table, inventory_id)` **globally**, so a bare `abs($c['id'])` could collide
with another config's row and make `insert()`'s upsert steal it.

### F-7 — `loadEnvFile()` overrides the process environment (NOT fixed — owner's call)

`core/config/app.php:25-58` `putenv()`s every `.env` key **unconditionally**, contrary to
standard dotenv semantics where real environment variables win.

This silently broke `tests/backfill/ledger_backfill_test.php`: the test `putenv()`s
`IMS_DATA_PATH` to a temp fixture-spec directory and passes it into the backfill
subprocess, but the subprocess's own bootstrap overwrote it with `.env`'s value, so the
fixture motherboard spec was "not found" and **all 20 assertions failed against correct
code**. Proved by removing `IMS_DATA_PATH` from the scratch `.env`: **ALL PASS**.

Same root cause is why `DB_NAME` cannot be steered by environment variable, which is why
this pass needed a disposable tree with its own `.env` at all.

**Deliberately not fixed here.** `app.php` is a production bootstrap that auto-deploys on
save; changing environment precedence in the middle of a verification pass is exactly the
unrelated risk a verification pass should not take. Recommended fix: only `putenv()` a key
when `getenv()` does not already return a value.

### F-7b — `finalize_command_test.php` hardcoded `created_by = 5` (FIXED, test-only)

No such `users.id` exists any more (live ids 2, 4, 38, 39, 43, 44, 46, 52), so every
fixture INSERT died on `fk_server_config_user` rather than testing anything. Now resolves
a real id at runtime.
